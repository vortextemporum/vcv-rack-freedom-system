## Key Architecture Decisions

**DSP Components:**
1. Waveform Generator (sine, triangle, saw, square, pulse, noise)
2. PolyBLEP anti-aliasing (for saw/square/pulse)
3. 2-Operator FM synthesis (linear phase modulation, YM2612-style)
4. Hard sync (Schmitt trigger-based phase reset)
5. Bit depth reduction (1-16 bits quantization)
6. Sample rate reduction (1kHz to full rate downsampling)
7. V/Oct pitch tracking (standard VCV Rack 1V/Oct)
8. Polyphonic voice management (16 channels)

**Module Variants:**
- **GenesisMono:** Monophonic (8 HP) - CPU-efficient for single-voice patches
- **GenesisPoly:** Polyphonic 16 voices (10 HP) - Full polyphonic support

**Implementation Strategy:**
- GenesisMono first (4 DSP phases): Basic oscillator → Anti-aliasing → FM/sync → Bit crushing
- GenesisPoly second (2 DSP phases): Convert to polyphonic → Optimize
- Shared DSP code between variants (only differ in polyphony handling)

**Risks & Mitigations:**
- PolyBLEP anti-aliasing: Reference Bogaudio VCO (open-source, proven implementation)
- FM synthesis: Test with YM2612 references (Streets of Rage, Sonic soundtracks)
- Polyphonic performance: Profile with 16 voices, optimize with SIMD if needed
